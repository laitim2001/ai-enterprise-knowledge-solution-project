## Keyword-mode recall 機制（揀 keyword 時請留意）

- eval retrieve top-5 chunk，逐個 `expected_answer_keywords` 做**大小寫不敏感 substring match**落 retrieved chunk_text；recall = 命中數 / 總數。
- **太多 keyword** → 有一個唔中就拉低 recall；**太空泛**（如 `AR`/`customer`）→ 無分辨力（邊段都中，驗唔到 retrieve 啱唔啱）。
- **甜蜜點 = 3-4 個喺*同一正確 section* 內共現嘅具體詞**（function 名 / 選單路徑 / section code 如 `AR01` `FA06`，呢啲都喺實際 chunk_text 出現且高分辨力）。
